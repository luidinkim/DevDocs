## 🏗️ 시스템 아키텍처

<Tabs items={['개요', '구성요소', '작동방식']}>
  <Tabs.Tab>
    ### SuperClaude 핵심 시스템
    
    #### 🎯 입력 시스템
    **사용자 명령을 받아 분석하는 첫 단계**
    - sc: 명령어 감지
    - 일반 명령어 처리
    - 컨텍스트 파악
    - 연결: 파서, 분석기
    
    #### 🎼 오케스트레이터
    **전체 시스템을 조율하는 지휘자**
    - 명령어 파싱
    - 복잡도 평가
    - 자동 활성화 결정
    - 연결: 페르소나, MCP 서버
    
    #### 🎭 페르소나 시스템
    **11개의 전문가 AI 성격**
    - Architect - 설계 전문가
    - Frontend - UI/UX 전문가
    - Backend - 서버 전문가
    - Security - 보안 전문가
    - 연결: 실행 엔진
    
    #### 🖥️ MCP 서버
    **외부 도구와 연동하는 서버**
    - Sequential - 체계적 사고
    - Context7 - 문서 검색
    - Magic - UI 생성
    - Playwright - 테스트
    - 연결: 실행 엔진
    
    #### ⚡ 실행 엔진
    **실제 작업을 수행하는 엔진**
    - 코드 생성
    - 파일 조작
    - 테스트 실행
    - 문서 생성
    - 연결: 출력
  </Tabs.Tab>

    **품질 검증**

## 🎯 주요 기능별 가이드

<Tabs items={['분석 기능', '구축 기능', '개선 기능', '자동화']}>
  <Tabs.Tab>
    ### 🔍 분석 기능 (sc:analyze)
    
    #### 분석 레벨
    
    **🟢 기본 분석**
    - 실행 시간: 5-10초
    - 토큰 사용: ~2K
    - 적합한 경우: 간단한 버그, 문법 오류
    
    **🔵 --think 모드**
    - 실행 시간: 30-60초
    - 토큰 사용: ~4K
    - Sequential MCP 자동 활성화
    - 적합한 경우: 복잡한 버그, 성능 문제
    
    **🟣 --think-hard 모드**
    - 실행 시간: 2-5분
    - 토큰 사용: ~10K
    - 모든 관련 MCP 서버 활성화
    - 적합한 경우: 리팩토링, 아키텍처 개선
    
    **🔴 --ultrathink 모드**
    - 실행 시간: 5-10분
    - 토큰 사용: ~32K
    - Wave 모드 자동 활성화
    - 적합한 경우: 레거시 현대화, 대규모 마이그레이션
  </Tabs.Tab>
  
  <Tabs.Tab>
    ### 🔨 구축 기능 (sc:build)
    
    #### 구축 프로세스
    
    **1. 프로젝트 분석**
    - package.json 분석
    - 디렉토리 구조 파악
    - 프레임워크 감지
    
    **2. 프레임워크 선택**
    - React/Next.js - 모던 웹
    - Vue/Nuxt - 프로그레시브
    - Express/Fastify - 백엔드
    
    **3. 컴포넌트 생성**
    - UI 컴포넌트 (Magic MCP)
    - API 엔드포인트
    - 데이터 모델
    
    **4. 시스템 통합**
    - 상태 관리 연결
    - 데이터베이스 연결
    - 라우팅 설정
    
    **5. 테스트**
    - 단위 테스트
    - E2E 테스트 (Playwright)
    - 성능 테스트
  </Tabs.Tab>
  
  <Tabs.Tab>

    **⚡ 성능 최적화**

    - 보안 헤더 설정
    
    **🔧 유지보수성**
    - 모듈화 강화
    - 문서화 개선
    - 테스트 커버리지 증가
    - 의존성 업데이트
  </Tabs.Tab>
  
  <Tabs.Tab>
    ### 🤖 자동화 기능
    
    #### 자동화 시스템
