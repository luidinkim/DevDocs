## MCP(Model Context Protocol) 서버란?

<Callout type="info" emoji="🚀">
**MCP 서버는 Claude Code의 능력을 무한대로 확장합니다!**  
외부 도구와 서비스를 Claude Code에 직접 연결하여 더 강력한 개발 환경을 만들 수 있습니다.
</Callout>

## 🌟 필수 MCP 서버 TOP 2

### 1. Sequential Thinking (최강 추천 ⭐⭐⭐⭐⭐)

#### 왜 Sequential이 게임 체인저인가?

Sequential은 Claude의 사고 과정을 **체계적**이고 **논리적**으로 만들어줍니다.
복잡한 문제를 단계별로 분해하고, 각 단계를 깊이 있게 분석합니다.

#### 핵심 기능
- 🧠 **구조화된 사고**: 문제를 논리적 단계로 분해
- 🔍 **심층 분석**: 각 단계별 상세 분석
- 📊 **체계적 접근**: 일관된 문제 해결 패턴
- 🎯 **정확한 결론**: 증거 기반 의사결정

#### 활성화 방법
```bash
# 자동 활성화 (복잡한 문제 감지 시)
/analyze --think-hard  # Sequential 자동 활성화

# 수동 활성화
/analyze --seq
/sc:analyze --sequential

# SuperClaude 명령어
sc:analyze --seq --ultrathink
```

### 2. Context7 (문서의 신 ⭐⭐⭐⭐⭐)

#### 왜 Context7이 필수인가?

Context7은 **모든 주요 라이브러리의 공식 문서**를 즉시 검색하고 활용합니다.
더 이상 구글링하거나 문서를 찾아 헤맬 필요가 없습니다!

#### 핵심 기능
- 📚 **공식 문서 접근**: 1000+ 라이브러리 문서
- 🔄 **실시간 업데이트**: 최신 버전 문서 자동 반영
- 🌍 **다국어 지원**: 한국어 문서 포함
- 💡 **예제 코드**: 실제 사용 가능한 코드 샘플

#### 활성화 방법
```bash
# 자동 활성화 (라이브러리 import 감지)
import React from 'react'  # Context7 자동 활성화

# 수동 활성화
/explain React hooks --c7
/sc:implement --context7

# SuperClaude 명령어
sc:explain "React hooks" --c7
```

#### 지원 라이브러리 (일부)
- **Frontend**: React, Vue, Angular, Svelte, Next.js
- **Backend**: Express, NestJS, FastAPI, Django
- **Database**: PostgreSQL, MongoDB, Redis
- **DevOps**: Docker, Kubernetes, Terraform
- **AI/ML**: TensorFlow, PyTorch, Scikit-learn

## 🎯 SuperClaude sc: 명령어 체계

### sc: 명령어란?

<Callout type="success" emoji="⚡">
**`sc:` 접두사는 SuperClaude의 고급 기능을 활성화합니다!**  
일반 명령어에 `sc:`를 붙이면 자동으로 최적의 MCP 서버와 페르소나가 활성화됩니다.
</Callout>

### 명령어 구조
```bash
sc:[명령어] [대상] [--플래그]
```

### 핵심 sc: 명령어

#### 개발 명령어
```bash
# 빌드 (자동 최적화)
sc:build my-app --next --typescript

# 구현 (MCP 자동 활성화)
sc:implement "사용자 인증" --with-tests

# 설계 (Sequential 활성화)
sc:design "마이크로서비스"
```

#### 분석 명령어
```bash
# 분석 (Sequential + Context7)
sc:analyze --ultrathink

# 문제 해결 (Sequential 필수)
sc:troubleshoot "성능 문제"

# 설명 (Context7 문서 활용)
sc:explain "React 18 신기능"
```

#### 품질 명령어
```bash
# 개선 (반복 실행)
sc:improve --loop --iterations 5

# 정리 (체계적 접근)
sc:cleanup --systematic

# 리팩토링 (패턴 적용)
sc:refactor --pattern factory
```

### sc: vs 일반 명령어 비교

| 일반 명령어 | sc: 명령어 | 차이점 |
|------------|-----------|--------|
| `/analyze` | `sc:analyze` | Sequential 자동 활성화, 심층 분석 |
| `/implement` | `sc:implement` | Context7 문서 활용, 베스트 프랙티스 |
| `/build` | `sc:build` | 프레임워크 최적화, 자동 설정 |
| `/test` | `sc:test` | 포괄적 테스트, 커버리지 보장 |

## 🎓 프로 팁

<Callout type="success" emoji="💡">
**MCP 서버 마스터가 되는 법:**

1. **Sequential은 필수**: 복잡한 문제는 무조건 `--seq`
2. **Context7은 기본**: 라이브러리 사용 시 항상 `--c7`
3. **조합이 힘**: 여러 MCP 서버를 동시에 활용
4. **sc: 명령어 습관화**: 일반 명령어보다 sc: 우선 사용
5. **자동 활성화 활용**: 수동보다 자동 활성화 신뢰
</Callout>
