## 첫 프로젝트: TODO 앱 → 프로덕션 (5분)

<Steps>
### 프로젝트 생성 (30초)
```bash
sc:build todo-app --next --typescript --tailwind
cd todo-app
```

### 데이터베이스 설정 (1분)
```bash
sc:implement prisma-schema --entities todo,user
sc:implement database-connection --sqlite
npx prisma migrate dev --name init
```

### CRUD API 구현 (1분)
```bash
sc:implement api/todos --crud --auth
sc:implement api/auth --jwt
```

### UI 컴포넌트 구현 (1분 30초)
```bash
sc:implement TodoList --type component --realtime
sc:implement TodoForm --type component --validation
sc:implement Layout --type component --responsive
```

### 테스트 및 배포 (1분)
```bash
sc:test --quick
sc:build --production
sc:implement vercel-config
vercel deploy
```
</Steps>

**결과**: https://your-todo-app.vercel.app 에서 실행 중인 앱

## 핵심 명령어 10개

### 1. `sc:build` - 프로젝트 생성
```bash
sc:build [name] --next --typescript --tailwind --prisma
```

### 2. `sc:implement` - 기능 구현
```bash
sc:implement [feature] --type component|api|service --with-tests
```

### 3. `sc:analyze` - 코드 분석
```bash
sc:analyze --performance --security --quality
```

### 4. `sc:test` - 테스트
```bash
sc:test --unit --integration --e2e --coverage 80
```

### 5. `sc:improve` - 코드 개선
```bash
sc:improve --refactor --optimize --clean
```

### 6. `sc:troubleshoot` - 문제 해결
```bash
sc:troubleshoot "error message" --debug --fix
```

### 7. `sc:design` - 아키텍처 설계
```bash
sc:design system --scalable --microservices
```

### 8. `sc:document` - 문서화
```bash
sc:document --api --readme --changelog
```

### 9. `sc:deploy` - 배포
```bash
sc:deploy --platform vercel|aws|gcp --production
```

### 10. `sc:monitor` - 모니터링
```bash
sc:monitor --errors --performance --usage
```

## MCP 서버 활용

### Sequential Thinking - 복잡한 문제 해결
```bash
# 성능 문제 체계적 해결
sc:troubleshoot "slow loading" --seq
# → 원인 분석 → 병목 지점 → 해결 방안 → 구현
```

### Context7 - 정확한 구현
```bash
# Next.js 14 App Router 정확한 구현
sc:implement app-router --c7
# → 공식 문서 기반 최신 패턴 적용
```
