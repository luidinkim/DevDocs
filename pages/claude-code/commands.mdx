# 📝 Claude Code 명령어 완벽 가이드

## 명령어 체계 이해하기

<Callout type="info" emoji="🎯">
Claude Code의 명령어는 **직관적**이고 **일관된** 패턴을 따릅니다.  
`/명령어 [대상] [--플래그]` 형식으로 구성됩니다.
</Callout>

## 핵심 명령어 카테고리

### 📊 명령어 구조
- **개발**: `/build`, `/implement`, `/design`
- **분석**: `/analyze`, `/troubleshoot`, `/explain`
- **품질**: `/improve`, `/cleanup`, `/refactor`
- **테스트**: `/test`
- **문서화**: `/document`
- **버전관리**: `/git`

## 🛠️ 개발 명령어

### `/build` - 프로젝트 빌더

**목적**: 새 프로젝트나 기능을 빌드합니다.

<Tabs items={['기본 사용법', '고급 옵션', '실전 예제']}>
  <Tabs.Tab>
    ```bash
    # 기본 빌드
    /build
    
    # 특정 타입 빌드
    /build frontend
    /build backend
    /build fullstack
    ```
  </Tabs.Tab>
  
  <Tabs.Tab>
    ```bash
    # 프레임워크 지정
    /build --framework react
    /build --framework express
    
    # 기술 스택 조합
    /build --react --typescript --tailwind
    /build --next --prisma --trpc
    
    # 최적화 옵션
    /build --optimization
    /build --production
    ```
  </Tabs.Tab>
  
  <Tabs.Tab>
    ```bash
    # Next.js 풀스택 앱
    /build my-saas --next --prisma --tailwind --typescript
    
    # Express API 서버
    /build api-server --express --mongodb --jwt
    
    # React Native 앱
    /build mobile-app --react-native --expo
    ```
  </Tabs.Tab>
</Tabs>

### `/implement` - 기능 구현

**목적**: 특정 기능이나 컴포넌트를 구현합니다.

```bash
# 기본 구현
/implement 로그인 기능

# 타입 지정
/implement 대시보드 --type component
/implement 사용자 API --type api
/implement 결제 시스템 --type feature

# 옵션 조합
/implement 채팅 기능 --type feature --with-tests --documentation
```

**구현 타입**:
- `component`: UI 컴포넌트
- `api`: API 엔드포인트
- `service`: 비즈니스 로직
- `feature`: 전체 기능
- `module`: 독립 모듈

### `/design` - 설계 및 아키텍처

**목적**: 시스템 설계와 아키텍처를 수립합니다.

```bash
# 시스템 설계
/design 마이크로서비스 아키텍처

# 데이터베이스 설계
/design database --entities user,product,order

# API 설계
/design REST API --openapi

# UI/UX 설계
/design 대시보드 UI --wireframe
```

## 🔍 분석 명령어

### `/analyze` - 코드 분석

**목적**: 코드베이스를 분석하고 인사이트를 제공합니다.

<Tabs items={['범위별', '초점별', '깊이별']}>
  <Tabs.Tab>
    ```bash
    # 범위 지정
    /analyze --scope file      # 단일 파일
    /analyze --scope module     # 모듈 단위
    /analyze --scope project    # 전체 프로젝트
    /analyze --scope system     # 시스템 전체
    ```
  </Tabs.Tab>
  
  <Tabs.Tab>
    ```bash
    # 초점 영역
    /analyze --focus performance    # 성능 분석
    /analyze --focus security       # 보안 분석
    /analyze --focus quality        # 품질 분석
    /analyze --focus architecture   # 구조 분석
    /analyze --focus dependencies   # 의존성 분석
    ```
  </Tabs.Tab>
  
  <Tabs.Tab>
    ```bash
    # 분석 깊이
    /analyze --think              # 기본 분석 (4K 토큰)
    /analyze --think-hard         # 심층 분석 (10K 토큰)
    /analyze --ultrathink         # 극한 분석 (32K 토큰)
    ```
  </Tabs.Tab>
</Tabs>

### `/troubleshoot` - 문제 해결

**목적**: 버그나 문제를 진단하고 해결합니다.

```bash
# 증상 기반 진단
/troubleshoot "로그인이 안 됩니다"

# 에러 메시지 분석
/troubleshoot "TypeError: Cannot read property 'x' of undefined"

# 성능 문제
/troubleshoot "페이지 로딩이 느립니다" --performance

# 보안 문제
/troubleshoot "SQL injection 가능성" --security
```

### `/explain` - 설명 및 교육

**목적**: 코드나 개념을 설명합니다.

```bash
# 코드 설명
/explain 이 함수가 뭐하는지

# 개념 설명
/explain React hooks --detailed

# 아키텍처 설명
/explain 우리 프로젝트 구조

# 초보자용 설명
/explain --eli5  # Explain Like I'm 5
```

## ✨ 품질 개선 명령어

### `/improve` - 코드 개선

**목적**: 기존 코드를 개선합니다.

```bash
# 전반적 개선
/improve

# 특정 측면 개선
/improve --focus performance   # 성능 최적화
/improve --focus readability   # 가독성 개선
/improve --focus security      # 보안 강화
/improve --focus accessibility # 접근성 개선

# 반복 개선
/improve --loop --iterations 5
```

### `/cleanup` - 코드 정리

**목적**: 기술 부채를 제거하고 코드를 정리합니다.

```bash
# 일반 정리
/cleanup

# 특정 정리
/cleanup --unused        # 미사용 코드 제거
/cleanup --duplicates    # 중복 제거
/cleanup --dependencies  # 의존성 정리
/cleanup --formatting    # 포맷팅 정리
```

### `/refactor` - 리팩토링

**목적**: 코드 구조를 개선합니다.

```bash
# 기본 리팩토링
/refactor

# 패턴 적용
/refactor --pattern singleton
/refactor --pattern factory
/refactor --pattern observer

# 대규모 리팩토링
/refactor --architectural
```

## 🔄 버전 관리 명령어

```bash
# 커밋
/git commit "기능 추가"

/git analyze-history
```

## 🚀 고급 플래그

### 성능 플래그

| 플래그 | 설명 | 사용 예시 |
|--------|------|-----------|
| `--uc` | 토큰 압축 (30-50% 절약) | `/analyze --uc` |
| `--delegate` | 병렬 처리 | `/analyze --delegate` |
| `--concurrency N` | 동시 작업 수 | `/build --concurrency 5` |
| `--cache` | 캐싱 활성화 | `/analyze --cache` |

### 사고 깊이 플래그

| 플래그 | 토큰 사용 | 적합한 경우 |
|--------|-----------|-------------|
| `--think` | ~4K | 간단한 분석 |
| `--think-hard` | ~10K | 복잡한 문제 |
| `--ultrathink` | ~32K | 시스템 재설계 |

### 안전성 플래그

| 플래그 | 설명 | 사용 시기 |
|--------|------|-----------|
| `--safe-mode` | 보수적 실행 | 프로덕션 코드 |
| `--validate` | 사전 검증 | 중요한 변경 |
| `--dry-run` | 시뮬레이션 | 실행 전 확인 |
